# Созданные файлы — FamilyTalk iOS

## 📦 Обзор

Создана базовая структура iOS-приложения для семейного мессенджера с упрощённой авторизацией, списком контактов и функционалом звонков.

---

## 📁 Структура файлов

### 🗂 Models/ — Модели данных
- **User.swift** — модель пользователя с online-статусом
- **AuthResponse.swift** — ответ авторизации и запрос
- **CallLog.swift** — история звонков, типы и статусы

### 🔧 Services/ — Бизнес-логика
- **NetworkService.swift** — базовый HTTP-клиент с Combine
- **AuthService.swift** — управление авторизацией и токенами
- **KeychainService.swift** — безопасное хранилище токенов
- **ContactsService.swift** — получение списка контактов и поиск
- **CallService.swift** — инициация и завершение звонков

### 🎨 ViewModels/ — MVVM
- **AuthViewModel.swift** — логика экрана авторизации
- **ContactsViewModel.swift** — управление списком контактов
- **CallViewModel.swift** — состояние и управление звонками

### 📱 Views/ — SwiftUI интерфейсы
- **AuthView.swift** — экран авторизации (телефон + имя)
- **ContactsView.swift** — список контактов с поиском
- **CallView.swift** — экран активного звонка
- **Components/AsyncImageView.swift** — компонент для аватаров

### 🛠 Utilities/
- **MockData.swift** — тестовые данные для разработки

### 📄 Корневые файлы
- **FamilyTalkApp.swift** — точка входа приложения
- **ContentView.swift** — оригинальный файл (не используется)
- **README.md** — руководство по проекту
- **API_REFERENCE.md** — документация REST API
- **ТЗ_Мессенджер.md** — техническое задание

## 🔌 API Endpoints (согласно ТЗ)

### Авторизация
- `POST /auth/simplified-login` — упрощённая авторизация
- `POST /auth/refresh` — обновление токенов

### Контакты
- `GET /users` — список всех контактов
- `GET /users/search?q=query` — поиск по никнейму
- `GET /users/:id` — информация о пользователе

### Звонки
- `POST /calls/initiate` — инициировать звонок
- `POST /calls/:id/end` — завершить звонок
- `GET /calls/history` — история звонков

## 🚀 Как запустить

1. **Откройте проект в Xcode:**
   ```bash
   open FamilyTalk.xcodeproj
   ```

2. **Укажите URL бэкенда:**
   - Откройте `Services/NetworkService.swift`
   - Замените `https://api.example.com` на реальный адрес

3. **Запустите приложение:**
   - Выберите симулятор или устройство
   - Нажмите `Cmd + R`

4. **Войдите:**
   - Введите номер: `+79991234567`
   - Введите имя: `Тестовый пользователь`
   - Нажмите "Войти"

## 📝 Примечания

- **Архитектура:** MVVM + Combine
- **Минимальная версия iOS:** 16.0+ (для SwiftUI features)
- **Зависимости:** Встроенные (URLSession, Combine, Keychain)
- **Следующая итерация:** Добавить Socket.io через SPM

---

## 🐛 Известные ограничения

1. **Нет реального WebRTC** — звонки симулируются
2. **Нет Socket.io** — real-time события не работают
3. **Нет обработки ошибок сети** — нужно добавить retry логику
4. **Нет кэша изображений** — используется встроенный AsyncImage (позже Kingfisher)
5. **Упрощённая авторизация** — нет SMS OTP (для разработки)
