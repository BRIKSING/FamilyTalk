# 🎯 Quick Start Guide — FamilyTalk

## 🚀 Быстрый старт (3 минуты)

### 1️⃣ Откройте проект
```bash
cd FamilyTalk
open FamilyTalk.xcodeproj
```

### 2️⃣ Добавьте файлы в Xcode

Все созданные файлы нужно добавить в проект через Xcode:

**File → Add Files to "FamilyTalk"...**

Или перетащите папки в левую панель Xcode.

#### Структура для добавления:

```
FamilyTalk/
├── Models/
│   ├── User.swift
│   ├── AuthResponse.swift
│   └── CallLog.swift
│
├── Services/
│   ├── NetworkService.swift
│   ├── AuthService.swift
│   ├── KeychainService.swift
│   ├── ContactsService.swift
│   └── CallService.swift
│
├── ViewModels/
│   ├── AuthViewModel.swift
│   ├── ContactsViewModel.swift
│   └── CallViewModel.swift
│
├── Views/
│   ├── AuthView.swift
│   ├── ContactsView.swift
│   ├── CallView.swift
│   └── Components/
│       └── AsyncImageView.swift
│
├── Utilities/
│   └── MockData.swift
│
└── FamilyTalkApp.swift (уже есть, заменён)
```

### 3️⃣ Настройте URL бэкенда

Откройте `Services/NetworkService.swift` и замените:

```swift
private let baseURL = "https://api.example.com"
```

На ваш реальный адрес бэкенда (или локальный для тестов):

```swift
private let baseURL = "http://localhost:3000"  // Для локальной разработки
// или
private let baseURL = "https://your-server.com"  // Для продакшена
```

### 4️⃣ Запустите приложение

1. Выберите симулятор (например, iPhone 15 Pro)
2. Нажмите **Cmd + R** или кнопку ▶️
3. Приложение запустится на экране авторизации

## 🧪 Тестирование без бэкенда

Если бэкенд ещё не готов, можно использовать mock-сервер:

### Вариант 1: Локальный JSON-сервер (Node.js)

```bash
npm install -g json-server
```

Создайте `db.json`:
```json
{
  "users": [
    {
      "id": "1",
      "phone": "+79991234567",
      "displayName": "Иван Иванов",
      "username": "ivan",
      "lastSeen": "2026-04-23T12:00:00Z"
    }
  ]
}
```

Запустите:
```bash
json-server --watch db.json --port 3000
```

### Вариант 2: Mock в коде

Измените `NetworkService.swift`:

```swift
#if DEBUG
// Возвращать mock-данные вместо реальных запросов
func request<T: Decodable>(...) -> AnyPublisher<T, NetworkError> {
    return Just(MockData.users as! T)
        .setFailureType(to: NetworkError.self)
        .eraseToAnyPublisher()
}
#endif
```

## 🐛 Частые проблемы

### 1. Ошибка компиляции "Cannot find type 'User'"

**Решение:** Убедитесь, что файлы добавлены в Target Membership.

Выберите файл → Inspector → Target Membership → ✅ FamilyTalk

---

### 2. Приложение крашится при авторизации

**Причина:** Бэкенд не отвечает или неверный URL.

**Решение:**
- Проверьте URL в `NetworkService.swift`
- Проверьте, запущен ли бэкенд
- Используйте mock-данные для теста

---

### 3. Контакты не загружаются

**Причина:** Токен не сохранился или истёк.

**Решение:**
- Выйдите и войдите заново
- Проверьте Keychain (в симуляторе может сброситься)
